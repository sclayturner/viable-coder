# Narrative Ledger

This working ledger tracks recurring scenes, questions, and callbacks for the playbook. It is not manuscript prose. Use it to keep literary structure intentional as outlines, chapters, modules, and exercises develop.

The governing rhythm is defined in `doctrine/reader-orientation-pattern.md`: Scene -> Tension -> Distinction -> Tool -> Return.

## Use Notes

- Add an anchor when a scene, question, or callback is likely to recur across sections.
- Keep anchors tied to doctrine, tools, modules, or signals. Do not track decorative anecdotes.
- Update the likely owner when a section outline changes.
- If an anchor becomes canonical terminology, update `doctrine/playbook-lexicon.md`.

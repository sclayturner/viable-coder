# Workflow: Draft Chapter

1. Read root `AGENTS.md` and `book/AGENTS.md`.
2. Read the relevant section outline in `book/outlines/`.
3. Read only the doctrine and research files needed for the chapter.
4. Identify which Clay source should carry the chapter: founder, engineer, personal life, or a deliberate combination.
5. If the chapter needs lived grounding, read `doctrine/voice/lived-answerability.md`; if considering the prior book, read `ai/source-guides/superhuman-2025-source-guide.md`.
6. Draft from the reader's situation toward a practical distinction, diagnostic, or move.
7. Keep research support out of the main prose unless it changes the reader's action.
8. Run `ai/evals/voice-checklist.md`, `ai/evals/structure-checklist.md`, and `ai/evals/doctrine-drift-checklist.md`.

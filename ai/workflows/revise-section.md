# Workflow: Revise Section

1. Read root `AGENTS.md` and `book/AGENTS.md`.
2. Read the target section outline and current section draft.
3. Identify the section's job in the four-part architecture.
4. Separate fixes into structure, doctrine, voice, examples, exercises, and evidence.
5. Make the smallest coherent revision.
6. Note any doctrine or module changes that should be propagated elsewhere.

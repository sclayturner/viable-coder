# Workflow: Revise Unit

1. Read root `AGENTS.md` and `curriculum/AGENTS.md`.
2. Read the target unit outline and current unit draft.
3. Identify the unit's job in the four-unit architecture.
4. Identify whether the unit is carried by founder, engineer, personal-life, or mixed Clay experience.
5. Separate fixes into structure, doctrine, voice, lived anchors, examples, exercises, and evidence.
6. Make the smallest coherent revision.
7. Note any doctrine, module, source-guide, or eval changes that should be propagated elsewhere.

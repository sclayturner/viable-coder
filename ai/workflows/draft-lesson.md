# Workflow: Draft Lesson

1. Read root `AGENTS.md` and `curriculum/AGENTS.md`.
2. Read the relevant unit outline in `curriculum/outlines/`.
3. Read only the doctrine and research files needed for the lesson.
4. Identify which Clay source should carry the lesson: founder, engineer, personal life, or a deliberate combination.
5. If the lesson needs lived grounding, read `doctrine/voice/lived-answerability.md`; if considering the prior book, read `ai/source-guides/superhuman-2025-source-guide.md`.
6. Draft from the builder's situation toward a practical distinction, diagnostic, or move.
7. Keep research support out of the main prose unless it changes the builder's action.
8. Run `ai/evals/voice-checklist.md`, `ai/evals/structure-checklist.md`, and `ai/evals/doctrine-drift-checklist.md`.

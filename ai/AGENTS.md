# AI Collaboration Instructions

Use this folder for collaboration machinery: source maps, workflows, evaluation checklists, and operating notes for AI-assisted authorship.

## Working Rules

- Keep AI process artifacts separate from book content.
- Use workflows for repeatable multi-step work.
- Use evals to check quality after drafting or restructuring.
- Use `ai/adlc-dogfooding.md` to keep repo work aligned with the ADLC instead of only describing it.
- Use `.codex/agents/` or `.claude/agents/` when an independent perspective is useful.
- When a workflow matures into reusable behavior, consider promoting it into both `.agents/skills/` and `.claude/skills/`.
- Update `ai/harness-map.md` whenever adding, removing, renaming, or intentionally diverging a harness artifact.
